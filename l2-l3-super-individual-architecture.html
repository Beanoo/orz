端到端交付全栈项目的超级个体架构

目标是把 PM 自然语言需求转换为 Conduiteg 实仓中的可验证交付分支。新增需求先拆解为通用 Atomic Skill DAG，Code Agent 负责代码理解和实现，Repository Metadata Index 只提供确定性仓库事实。

AI 交付内核

Agent / Skill

Clarification Agent

识别歧义、矛盾、角色、边界、验收缺口，必要时阻断直接生成。

Skill Registry / Skill DAG

需求拆为 Atomic Skill 组合；Recipe 只作为可选组合模板，不做一需求一 Skill。

Code Agent / Explorer / Architect

阅读源码、确定模块边界、设计实现方案、生成和修复代码。

Repository Metadata Index

只提供确定性仓库事实：文件树、routes、models、controllers、services、tests、scripts。

交付与验证

Real Repo

Model Adapter

默认调用本地 codex，支持通过环境变量配置命令、模型和超时。

Cross-stack Consistency Engine

生成并验证字段从数据层到 API、前端、测试的传播合约。

Git Delivery Service

创建交付分支，写入 Conduiteg，运行验证并生成 commit 证据。

用户与系统入口 编排与交付 Agent / Skill 代码理解与仓库事实 持久状态

Atomic Skill DAG

clarify-requirement 歧义、矛盾、角色、边界、验收缺口。
map-codebase-context 结合仓库事实和源码阅读定位关键模块。
add-data-field 模型字段、迁移、默认值、兼容策略。
update-api-contract 请求参数、响应序列化、错误处理。
update-frontend-form 表单状态、校验、提交载荷、编辑回填。
update-frontend-rendering 列表、详情、空态和兼容展示。
generate-contract-tests 跨栈契约测试和回归覆盖。
run-verification lint、unit、build、smoke check。
review-change 一致性、风险、缺失测试、回归检查。
memory-writeback 沉淀可复用决策、文件路径和验证证据。

Repository Metadata Index 边界

它负责 确定性事实索引、缓存、文件图谱、脚本、路由、模型、控制器、组件、测试映射。
它不负责 需求理解、架构决策、代码生成、最终模块边界判断。
Code Explorer 基于索引结果继续读源码，输出调用链、关键文件和现有模式。
Code Architect 结合 DSL、Explorer 结果和一致性合约设计实现方案。

L2 示例链路：Article.coverImage

1 需求拆解 add-field、persist-data、expose-api、edit-form、render-view、test-contract。
2 Skill DAG 选择通用 Atomic Skill；Recipe 只作为可选加速模板。
3 模块定位 models、migrations、controllers、services、editor、list、detail、tests。
4 一致性合约 验证字段从后端源头传播到 API、前端表单、展示和测试。
5 生成与写入 本地 codex 生成 patch，Git Delivery Service 写入 Conduiteg 分支。
6 验证与记忆 运行真实验证命令，提交 commit，并沉淀历史交付记忆。
